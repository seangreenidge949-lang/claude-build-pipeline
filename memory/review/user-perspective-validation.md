写报告/UI 相关代码时，技术验证通过后必须做用户视角验证。

**Why:** 2026-03-21 UserSim session 用户连续反馈 5+ 个产品问题：
- 原声没价值：采集了 LLM 加工后的 overall_impression（总结），丢弃了场景级 emotional_trajectory/problems_encountered/confusion_points（第一现场数据）
- 看不到每用户详情：只有聚合视图，无法追踪单个用户的完整旅程
- 场景名截断到 12 字信息严重丢失
- Chat 功能做了但报告页没有任何入口能跳转
- operation_path 等丰富数据存在但前端完全没展示

**How to apply — 用户视角 Checklist:**
1. 这个数据对用户有什么价值？原声要有具体使用问题，不是空洞总结
2. 用户看到这个能做什么下一步动作？有详情要能展开，有用户要能对话
3. 信息粒度够不够？聚合之外是否需要明细？截断是否丢失关键语义？
4. 数据有没有被浪费？后端已有的丰富字段前端是否都展示了？
5. 功能链路是否打通？做了 Chat 就要在报告页有入口，做了详情就要有展开交互
