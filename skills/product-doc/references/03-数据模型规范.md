# 数据模型规范

> 用 code block 定义实体，简洁但完整。

---

## 格式

直接用目标语言的 class/struct 定义，字段名+类型+注释：

```dart
class MeetingRecord {
  String id;                    // UUID, 主键
  String rawText;               // 原始会议记录文本
  String title;                 // AI 提取的会议主题, ≤50字
  String? tldr;                 // 一句话摘要, 为空时不展示
  MoodType overallMood;         // consensus/mixed/controversial, 默认 mixed
  List<Discussion> discussions; // 讨论点列表, 空列表时显示空态
  List<ActionItem> actionItems; // 待办事项, 空列表时显示空态
  DateTime createdAt;           // 本地时间, 自动生成
}
```

## 必须标注的信息

在注释中写明：

| 信息 | 何时标注 |
|------|---------|
| 默认值 | 有默认值时（如 `默认 draft`） |
| 空值行为 | 字段可选时（如 `为空时不展示`、`为空时显示默认头像`） |
| 枚举值 | 枚举字段必须穷举（如 `draft/paid/shipped`） |
| 长度限制 | 有上限时（如 `≤50字`） |

## 字段类型约束

| 规则 | 说明 |
|------|------|
| 金额/价格 | 禁止 Float/Double，用 Decimal 或整型存分 |
| 枚举 | 穷举所有值 |
| 时间 | 标注时区（UTC/本地）|
| 外键 | 声明删除策略：CASCADE / SET NULL / RESTRICT |

## 数据映射信号词

文档中出现以下词时，必须输出完整映射表（不能只给示例）：

| 信号词 | 必须交付 |
|--------|---------|
| "文案根据XX变化" | {条件值: 对应文案} 全量对照表 |
| "颜色/图标随XX变化" | {条件值: 颜色值/图标名} 全量对照表 |
| "格式化展示" | 格式化规则 |
| "排序" | 排序维度 + 升降序 |
